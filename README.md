## Учебный проект по написанию микросервисов
### Система состоит из трёх микросервисов (основной сервис обработки заказов, сервис-заглушка для API с I/0-bound нашрузкой, сервис уведомлений по email). 
### Основной сервис принимает заказы через REST API, сохраняет их в БД (заказ и задачу для него) и обрабатывает асинхронно через polling. Каждое какое-то заданное время Poller опрашивает БД с помощью SELECT FOR UPDATE SKIP LOCKED и забирает задачи, готовые к обработке. В рамках одной транзакции выполняются параллельные запросы к трём внешним API (инвентаризация → цены → доставка). При успехе публикуется событие в Kafka, которое потребляет email-сервис для отправки уведомлений. При неуспехе делает статус retry, чтобы задачу попробовала выполниться ещё раз. Все внешние зависимости эмулируются stub-сервисом с I/0-bound нагрузкой для локальной разработки и тестирования.
Email лежит локально проверять на 8025 порту
